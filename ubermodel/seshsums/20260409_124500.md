## Change 1: Parameter prefix convention

All parameter names and computed output keys now carry a two-letter prefix:

| Prefix | Meaning | Settable by user? |
|---|---|---|
| `I_` | Primary input — physical constant or experimental measurement | Yes, main knobs |
| `B_` | Background value — scenario engineering/economic parameter | Rarely; stable defaults |
| `P_` | Partial result — intermediate computed value | Never |
| `O_` | Primary output — final reported result | Never |

### Motivation

The original code was a flat namespace where physical constants, scenario knobs,
intermediate derivations, and final answers all looked identical.  The prefixes
make the role of every value unambiguous at a glance — in the code, in the TSV
headers, and on the CLI.

### What was renamed

**I_ (5 values)** — the primary inputs that are scenario-independent:
`I_ch4_per_kwy`, `I_co2e_ratio`, `I_ch4_reference_concentration_ppb`,
`I_credit_price_per_tonne_ch4`, `I_credit_price_per_tonne_co2e`

**B_ (24 values)** — all scenario setting parameters across ship, solar,
landfill, and troposphere bundles.  Examples: `B_ship_days_per_year`,
`B_solar_nameplate_mw`, `B_landfill_construction_premium`, `B_years_to_run`.

**P_ (partial results returned from compute functions)** — intermediate values
that appear in the TSV for traceability but are never user-settable.  Examples:
`P_ship_energy_used_kwy`, `P_ship_energy_cost_per_year`, `P_fleet_real_power_mw`,
`P_solar_available_kw`, `P_solar_annual_capex`, `P_landfill_concentration_factor`,
`P_wy_to_remove_1ppm_troposphere`.

**O_ (primary outputs)** — the final answers shown in the terminal and TSV.
Examples: `O_ship_ch4_converted_tonnes`, `O_solar_net_ch4`,
`O_landfill_revenue_ch4`, `O_nukes_needed`, `O_annual_cost`.

### CLI impact

Overrides on the command line now require the full prefixed name:

```bash
--I_ch4_per_kwy 4.5
--B_solar_nameplate_mw '[50,200,50]'
```

Attempting to override a `P_` or `O_` name will produce a warning
("not a recognised parameter name") since those keys exist only in the
output dict, not the input dict.

### Code impact

- `PRIMARY_INPUTS` dict keys all gained `I_` prefix
- `SETTINGS` dict keys all gained `B_` prefix
- All four `compute_*()` functions updated: parameter lookups use `p['I_*']`
  and `p['B_*']`; returned dict keys use `P_*` or `O_*`
- `print_results()` updated to reference the new output key names
- No logic was changed; only names

## Change 2: README updates

Three additions to `README.md`:

### "Scenarios and --setting" section expanded

Added a prose explanation of what `--setting all` actually does, answering the
question "I don't understand what 'all' does?":

- `all` merges all four scenario parameter bundles into one flat dict
- It runs all four compute functions and collects all outputs
- The TSV contains every parameter and output column in one row
- Single-scenario settings keep the TSV clean for focused sweeps
- A parameter override under `--setting all` propagates to every scenario
  that uses that parameter

### Parameter naming convention section added

New section documents the `I_/B_/P_/O_` prefix system with a table explaining
each tier, notes on which are user-settable, and updated CLI examples using the
prefixed names.

### Complete rename table added

Full before/after table listing every old parameter name and its new prefixed
equivalent, grouped by tier.  Partial results and outputs are described by
example rather than exhaustive enumeration since they are never user-facing
inputs.

### Parameter tables updated

All individual parameter tables updated to use the new `I_*` and `B_*` names.
Output listings updated to use `O_*` names.  Model notes updated to reference
`I_ch4_per_kwy` and `I_ch4_reference_concentration_ppb` by their new names.
